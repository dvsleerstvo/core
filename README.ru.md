# DV SLEERSTVO (ДВ СЛЕЕРСТВО) - Geometry Dash Demonlist

[English](README.md) | [Русский](README.ru.md)

Специализированная таблица лидеров и система учета рекордов для игроков Geometry Dash на Дальнем Востоке России (Приморский край, Хабаровский край и т.д.).

## 🏗 Технологический стек

### Бэкенд
- **Фреймворк**: Django 5.2+ (Python 3.13+)
- **API**: Django REST Framework (DRF)
- **База данных**: PostgreSQL
- **Кэширование**: Redis
- **Авторизация**: Discord OAuth2 (через `django-allauth`)
- **UI Фреймворк (Админка)**: `django-unfold` (современный интерфейс администратора)
- **Управление зависимостями**: `uv`

### Фронтенд
- **Фреймворк**: Svelte 5 (с использованием SvelteKit 2)
- **Стилизация**: Vanilla CSS
- **Карты**: Leaflet (для распределения игроков по регионам)
- **Язык**: TypeScript

## 📂 Структура проекта

- `/demonlist`: Основное приложение Django, содержащее модели, API представления и бизнес-логику.
- `/frontend`: Приложение SvelteKit.
- `/core`: Конфигурация проекта Django.
- `/discordbot`: Discord-бот для уведомлений о рекордах и синхронизации.
- `/static`: Статические файлы.
- `/templates`: Пользовательские шаблоны Django (в основном для переопределения админки).
- `/maptest`: Экспериментальный функционал карт.

## 🛠 Основные модели и логика

- **Level**: Уровни GD с рейтингами (ПК/Мобайл), очками и минимальными требованиями к прогрессу.
- **User**: Профили игроков, привязанные к аккаунтам GD и Discord. Отслеживает очки и самые сложные прохождения.
- **Victor**: Подтвержденные прохождения или значительный прогресс на уровнях.
- **RecordRequest**: Система заявок для подтверждения новых рекордов.
- **DiscordLink**: Связь между пользователем Django и профилем `demonlist.User`.

## 🚀 Запуск проекта

Проект полностью докеризирован для удобства разработки.

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/dvsleerstvo/core.git
   cd core
   ```

2. Создайте файл `.env` на основе примера (`.env.example`) и укажите необходимые переменные:
   - `SECRET_KEY`
   - Учетные данные Discord-бота (`BOT_TOKEN`, `BOT_API_SECRET`, `REQUEST_CHANNEL_ID`)
   - *Примечание: Учетные данные базы данных уже прописаны по умолчанию для локального запуска.*

3. Запустите приложение с помощью Docker Compose:
   ```bash
   docker compose up --build
   ```
   *Примечание: По умолчанию контейнер с Discord-ботом **не** запускается. Если вы хотите запустить бота вместе с проектом, используйте профиль `bot`:*
   ```bash
   docker compose --profile bot up --build
   ```
